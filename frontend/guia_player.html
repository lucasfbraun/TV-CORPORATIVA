🪟 2. Configuração no Windows (PC / Mini-PC)

2.1 Pré-requisitos

  • Windows 10 ou 11
  • Google Chrome instalado
  • Conexão com a rede local da empresa

2.2 Configurar o Chrome em modo Kiosk (tela cheia)

O modo kiosk abre o Chrome já em tela cheia, sem barra de endereços ou botões, ideal para TVs.

  1. 1
    Crie um atalho do Chrome na Área de Trabalho: clique com o botão direito → Novo → Atalho.
  2. 2
    No campo de local do atalho, insira o comando abaixo (ajuste o IP e porta conforme seu servidor): 
    "C:\Program Files\Google\Chrome\Application\chrome.exe" --kiosk http://[IP-SERVIDOR]:8080/display.html --no-first-run --disable-infobars --autoplay-policy=no-user-gesture-required

    Se usar arquivo local (sem servidor), substitua a URL por:

    "C:\Users\[usuario]\...\tv corporativa\display.html"
  3. 3
    Nomeie o atalho "TV Corporativa" e clique em Concluir.
  4. 4
    Teste o atalho. A TV deve abrir em tela cheia sem bordas. Pressione Alt+F4 para sair do modo kiosk quando necessário.

2.3 Iniciar automaticamente com o Windows

  1. 1
    Pressione Win + R, digite shell:startup e pressione Enter. A pasta de Inicialização do Windows abrirá.
  2. 2
    Copie o atalho "TV Corporativa" para essa pasta. Agora ele abrirá automaticamente ao ligar o computador.
  3. 3
    Configure o Windows para fazer login automático: Win + R → netplwiz → desmarque "Os usuários devem digitar um nome de usuário e senha" → OK.

2.4 Configurar desligamento e reinício automático

No Agendador de Tarefas do Windows, crie duas tarefas:

TarefaHorárioAção
Ligar TV (reiniciar) Seg–Sex, 07:30 shutdown /r /t 0
Desligar TV Seg–Sex, 18:00 shutdown /s /t 0
✅
Dica: Configure também o BIOS do mini-PC para ligar automaticamente quando a energia for restaurada. Procure por "Power On After AC Loss" ou "Restore on AC Power Loss" nas configurações BIOS e defina como "Power On".

🍓 3. Configuração no Raspberry Pi

3.1 Pré-requisitos

  • Raspberry Pi 4 (2GB RAM ou mais)
  • Cartão microSD 32GB (Classe 10)
  • Raspberry Pi OS (64-bit, Bullseye ou Bookworm)

3.2 Instalação do Chromium em modo kiosk

Execute os comandos abaixo no terminal do Raspberry Pi:

# Atualizar o sistema
sudo apt update && sudo apt upgrade -y

# Instalar Chromium e utilitários necessários
sudo apt install -y chromium-browser unclutter xdotool

# Desativar protetor de tela e suspensão de tela
sudo raspi-config nonint do_blanking 1

3.3 Configurar autostart do kiosk

# Criar arquivo de autostart
mkdir -p ~/.config/autostart
nano ~/.config/autostart/tv-kiosk.desktop

Insira o seguinte conteúdo (substitua o IP):

[Desktop Entry]
Type=Application
Name=TV Corporativa
Exec=chromium-browser --kiosk --noerrdialogs --disable-infobars \
  --autoplay-policy=no-user-gesture-required \
  http://[IP-SERVIDOR]:8080/display.html
X-GNOME-Autostart-enabled=true

3.4 Manter a tela sempre ligada

# Criar script para manter tela ativa
nano ~/no-sleep.sh
#!/bin/bash
xset s off
xset -dpms
xset s noblank
chmod +x ~/no-sleep.sh

# Adicionar ao autostart
echo "Exec=bash /home/pi/no-sleep.sh" >> ~/.config/autostart/tv-kiosk.desktop
⚠️
Temperatura: Em ambientes com alta temperatura (chão de fábrica), use um case com dissipador de calor ou ventilador para o Raspberry Pi. A temperatura ideal de operação é abaixo de 70°C.

🤖 4. Configuração em Android Box / Smart TV

4.1 Instalar o Google Chrome

  1. 1
    Abra a Google Play Store no dispositivo.
  2. 2
    Pesquise "Google Chrome" e instale.
  3. 3
    Abra o Chrome e acesse http://[IP-SERVIDOR]:8080/display.html.
  4. 4
    Pressione F11 ou ative o modo tela cheia pelo menu do Chrome.

4.2 Iniciar automaticamente

Use um app de autostart para Android, como o AutoStart – No Root (disponível na Play Store), para abrir o Chrome automaticamente ao ligar o dispositivo.

💡
Smart TV Samsung/LG: Use o navegador nativo da TV e configure-o como a "Home Screen" nas configurações de mídia da TV. Consulte o manual da sua TV para detalhes.

🌐 5. Configuração de Rede

5.1 Topologia recomendada

O servidor deve estar em um computador fixo e acessível para todos os players via rede local:


  [Computador do Admin]
        │  admin.html
        │  server.py (:8080)
        │
  ──────┼────────────────── Rede Local (192.168.x.x)
        │
   ┌────┴──────────────────────────────────────┐
   │                                           │
[TV 1]                                      [TV 2..20]
http://192.168.1.10:8080/display.html       (mesma URL)

5.2 IP fixo para o servidor

Configure IP fixo no computador que roda o server.py para que a URL das TVs não mude:

  1. 1
    Painel de Controle → Central de Rede → Alterar as configurações do adaptador.
  2. 2
    Clique com botão direito na conexão → Propriedades → Protocolo TCP/IPv4.
  3. 3
    Selecione "Usar o seguinte endereço IP" e defina um IP fixo (ex: 192.168.1.10).
⚠️
Alternativa: Configure uma reserva de DHCP no roteador para que o servidor sempre receba o mesmo IP. Consulte o suporte de TI para configurar isso no roteador da empresa.

🔧 6. Manutenção e Troubleshooting

ProblemaCausa provávelSolução
Tela em branco ao ligar Chrome não iniciou automaticamente Verifique a pasta de Inicialização do Windows ou o arquivo autostart do RPi. Reinicie o dispositivo.
Conteúdo desatualizado Servidor offline ou sem conexão Verifique se o server.py está rodando. Abra http://[IP]:8080 em outro computador para testar.
TV trava após horas Memória RAM cheia (especialmente RPi) Configure reinicialização automática diária (00:00) via Agendador de Tarefas ou cron.
Slides não passam Erro no JavaScript Pressione F12 no Chrome → Console → copie o erro e contate o TI.
Imagem distorcida Resolução incorreta Configure a resolução do PC/RPi para 1920×1080 (Full HD) nas configurações de vídeo.
